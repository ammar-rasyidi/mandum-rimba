# Wildlife distribution pipeline

Builds the map's **Wildlife Distribution** layer (`species-distribution.geojson`)
and the campaign / KTP "nearest species" index (`wildlife-points.json`) from
public data. This is an **offline** build, run occasionally, not part of the
nightly server pipeline. See [../../DATA-FLOW.md](../../DATA-FLOW.md) for context.

## What it produces

| Output | Copied to | Used by |
| --- | --- | --- |
| `species-distribution.geojson` | `apps/web/public/data/` | the map's Wildlife Distribution layer |
| `species-names.json` | `apps/web/src/data/` | common names in the tooltip / cards |
| `wildlife-points.json` | `apps/web/src/data/` | "nearest threatened species" (campaign + KTP) |

## Method (why it looks the way it does)

1. **Occurrences**, GBIF records for threatened + flagship/endemic species
   across all classes, pulled **per region** so under-surveyed areas (e.g. Papua)
   are represented, not just observer hotspots.
2. **Status**, each species' **IUCN** category; we keep conservation-relevant
   species (threatened everywhere; plus all native non-bird wildlife in the
   highly-endemic east, Wallacea + Papua, since nearly all of it is endemic).
3. **Habitat weighting**, occurrence density is weighted by **ESA WorldCover**
   natural-habitat cover (forest/savanna/wetland/shrub/mangrove) and city points
   are dropped, so areas follow real habitat, not cities or sampling bias.
4. **Per-island contours**, contoured **per island** so species stay on their
   actual island (a Sumatran tiger never bleeds into Java), with smooth edges.
5. **Documented-range markers**, a few species whose public coordinates are
   withheld for protection (e.g. Sumatran rhino) are placed at their documented
   protected areas, clearly flagged as documented range, not field records
   (see `markers.json`).

## Run it

Requires Python with `numpy`, `matplotlib`, `rasterio`, `shapely`. Run each step
from this directory; intermediate artifacts (gitignored) land here, then copy
the three outputs into the app. Steps are roughly in order:

```bash
python pull_cache.py          # GBIF occurrences            -> raw_points.json
python notable_pull.py        # flagship + endemic by key   -> raw_notable.json
python fetch_iucn.py          # IUCN category per species   -> iucn.json
python build_natural_mask.py  # ESA WorldCover habitat grid -> natural_grid.npy
python contour.py             # the layer                   -> species-distribution.geojson
python names.py               # GBIF common names           -> species-names.json
python gen_wildlife.py        # nearest-species index       -> wildlife-points.json

cp species-distribution.geojson ../../apps/web/public/data/
cp species-names.json wildlife-points.json ../../apps/web/src/data/
```

`contour.py` takes optional `sigma quantile min_area` args to tune smoothing vs
coverage (defaults are tuned for the current look).

## Committed inputs (small, curated/derived)

| File | What |
| --- | --- |
| `flagship.json` | 30 flagship species: scientific + Indonesian name + GBIF key (from `apps/api/.../flagship-species.ts`) |
| `flagship_regions.json` | flagship species → island(s), so they stay in range |
| `birds_matched.json` | endemic-bird GBIF taxon keys |
| `markers.json` | documented-range markers (rhino, dolphin sites) |
| `urban_id.json` | Natural Earth urban polygons (Indonesia), to drop city points |

The large intermediate caches (`raw_points.json`, `raw_notable.json`,
`iucn.json`, `natural_grid.npy`) are regenerated by the scripts and **not**
committed.

## Sources

GBIF (CC BY, occurrences + taxonomy + vernacular names) · IUCN Red List (status)
· ESA WorldCover 2021 (CC BY 4.0, habitat) · Natural Earth (public domain, urban
areas). IUCN authoritative range polygons are licence-restricted and not used.
